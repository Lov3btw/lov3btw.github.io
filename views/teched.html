Project Overview

The product:

I are creating an app, as well as a responsive mobile, and desktop website that tutors adults who want to learn technical skills. They also want to offer a collective for those who want to stay current on new advancements and new trends in the tech community.

Project duration:

January-Feburary 2023

The problem:

Adults who want to learn new skills in the technical field can be intimidated about where to get started and want a comfortable experience they can access in their home or on the go, depending on their needs and lifestyle.

The goal:

To offer a pleasant and educational experience for users who want to learn at any time in any space and offer those who just want to stay current on new advancements and trends in technology an all in one experience.

My role:

Lead UX designer, UX researcher

Responsibilities:

User research, Wire-framing, Prototyping, Lead Designer

An unmoderated usability study was conducted remotely on February 9th, 2023. We took a qualitative approach in our primary interviews. The results of those interviews showed that users wanted an internal communication system and a way to save or like content, courses, and tutors to their profile as well as see ratings and featured content. There were a few user flow and feature location issues that we needed to address. We conducted a usability study followed by a Systems Usability Scale.

User Pain Points

Save Feature

Users want to be able to save tutors, courses and content.

Messaging System

Users want to be able to communicate with tutors and peers internally.

Icons/feature location

It was noted that certain features should be relocated and the menu icon changed to a settings icon which would be a better representation.

Research

My target audience are users getting started later in life and user who want to continue learning and developing skills.

Paper Wireframes:

Based on the competitive audit, I saw a need to create an easy experience that was warm and friendly. Putting an emphasis on being informative without being overwhelming to new learners and making them feel comfortable became a priority. Including an available tutor profile page with information about the tutor would offer a more personalized experience. If the user felt they knew the tutor they were booking it could eliminate any apprehension the learner may have and allow them to focus on learning. Simplified navigation offers a clean homepage and information the user may not necessarily need right away can be found there when needed.

Low Fidelity Prototype

The low-fidelity prototype was designed to test the user flow of the checkout process including completing an order, as well as locating and accessing the accessibility and customer service features.

Round 1 findings

Results of the Usability study showed users wanted a save feature as well as a way to message tutors or peers through the site or app

Users want to be able to communicate with tutors and peers internally.

It was noted that certain features should be relocated and the menu icon changed to a settings icon which would be a better representation.

Bookmarks were added for users to save favorite tutors to their user profile. I’m ideating an option to receive notifications when a favorited tutor has posted new content or has immediate openings for sessions.

Offering an easy and comfortable experience we chose to include words of encouragement on the homepage as well as images of people to make for a more welcoming experience. You can see we changed the menu icon to a settings icon to more appropriately reflect its contents, as well as rearranged the icons.

Adding bookmarks to the tutor profile page would allow learners to read about the tutor before bookmarking for future use. Also pre-recorded courses by that tutor can be bookmarked for later viewing. I also added an “Add to Calendar” option in the booking confirmation modul for convenience.

High Fidelity Prototype

I added the option to bookmark content and tutors as well as reorganized the icons and moved some features to more appropriate areas based on user feedback.
